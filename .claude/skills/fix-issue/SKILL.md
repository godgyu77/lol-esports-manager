# Fix Issue Skill

> 이슈를 분석하고 체계적으로 수정하는 스킬.
> 디버깅 상세 프로세스는 `agents/debugger.md` 참조.

---

## 언제 사용하나

- 버그/이슈 수정 요청 시
- 에러 발생 시 root cause 분석이 필요할 때

---

## 워크플로우

1. **에러 수집** — 에러 메시지(원문), 스택 트레이스, 재현 조건 파악
2. **원인 분석** — 가설 수립 → 코드/로그 검증 → root cause 확정 (→ `agents/debugger.md` 프로세스)
3. **최소 수정** — 원인만 정확히 수정 (과도한 리팩토링 금지)
4. **검증** — 빌드/타입체크 + 회귀 위험 지점 확인
5. **결과 보고** — 변경 파일, 수정 내용, 남은 작업

---

## 출력 형식

```
## Root Cause
한 줄 요약

## Evidence
원인을 뒷받침하는 코드/로그

## Fix
수정한 코드 diff

## Prevention
재발 방지책
```
