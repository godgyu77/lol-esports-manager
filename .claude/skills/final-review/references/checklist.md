# Final Review Checklist

> 출처: lhg checklist.md

---

## 1. 범위

- [ ] 요청 범위 밖 파일 수정이 없는가
- [ ] 무관한 리팩터링/리네이밍이 없는가

## 2. 코드 품질 규칙

- [ ] 분기/맥락이 과도하게 섞이지 않았는가
- [ ] 이름/타입/반환 형태가 예측 가능한가
- [ ] 함께 수정될 코드가 떨어져 있지 않은가
- [ ] 변경 영향 범위를 통제했는가

## 3. 프로젝트 규칙

- [ ] 구조 변경이 필요한 경우 사유가 명확한가
- [ ] 프로젝트 전용 API / 상태 관리 규칙을 따르는가
- [ ] 목업/테스트 플로우가 유지되는가
- [ ] 프로젝트 프레임워크 설계 원칙을 유지하는가

## 4. 성능 규칙

- [ ] 워터폴 위험을 먼저 제거했는가
- [ ] 불필요한 번들 증가가 없는가
- [ ] `useMemo/useCallback`을 과용하지 않았는가

## 5. 보안 규칙

- [ ] 사용자 입력에 대한 검증/sanitize가 있는가
- [ ] 민감 정보가 클라이언트/로그에 노출되지 않는가
- [ ] SQL 파라미터 바인딩을 사용하는가

## 6. 검증/보고

- [ ] 타입체크/린트/테스트 결과를 확인했는가
- [ ] 미실행 검증이 있으면 이유를 적었는가
- [ ] 변경 파일과 핵심 의도를 보고했는가
