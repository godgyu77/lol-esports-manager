# Final Review Skill

> 구현 완료 후 최종 승인 전에 품질 게이트를 통과시키는 용도다.
> 출처: lhg final-review 기반

---

## 언제 사용하나

- 모든 코드 수정 작업의 마지막 단계

---

## 체크 절차

### 1. 범위 준수 점검

- 요청 외 수정 여부 확인
- Action: 변경 파일 목록을 요청 범위와 대조한다.

### 2. 코드 품질 4원칙 점검

- 가독성 / 예측 가능성 / 응집도 / 결합도
- Action: 항목별 위반 여부를 `있음/없음`으로 기록한다.

### 3. 스택 규칙 점검

- API / 상태 / 목업 / 구조 규칙 위반 여부
- Action: 프로젝트 오버레이 규칙 위반 여부를 별도로 확인한다.

### 4. 성능 우선순위 점검

- 워터폴 / 번들 / 데이터흐름 / 리렌더 순서 준수
- Action: 개선이 리렌더 최적화에만 치우치지 않았는지 확인한다.

### 5. 보안 규칙 점검

- XSS / 시크릿 노출 / SQL 인젝션 / 입력 검증
- Action: `security-guidelines.md` 체크리스트 항목을 순서대로 확인한다.

### 6. 검증 결과 점검

- 타입체크 / 린트 / 테스트 실행 여부 및 결과
- Action: 미실행 항목이 있으면 이유를 명시한다.

### 7. 보고 품질 점검

- 변경 파일 목록, 변경 이유, 미실행 항목, 리스크 보고
- Action: 최종 보고서 형식을 맞추고 누락 항목이 없도록 확인한다.
