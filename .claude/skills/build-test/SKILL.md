# Build & Test Skill

> 코드 변경 후 빌드/테스트를 실행하여 검증하는 스킬.
> 출처: jtw build-test 기반 + 프론트엔드 검증 추가

---

## 언제 사용하나

- 코드 변경 후 검증이 필요할 때
- CI/CD 전 로컬 검증 시
- 수정 사항의 회귀 여부를 확인할 때

---

## Java / Spring Boot 프로젝트

### 실행 순서

1. `mvn clean compile` — 컴파일 에러 확인
2. 컴파일 성공 시 `mvn test` — 테스트 실행
3. 실패한 테스트가 있으면 에러 메시지 분석 후 원인 요약
4. 전체 결과 리포트 (성공/실패/스킵 수)

### 특정 테스트만 실행

- `mvn test -Dtest={테스트클래스명}`
- `mvn test -Dtest={클래스명}#{메서드명}`

---

## React / TypeScript 프로젝트

### 실행 순서

1. `tsc --noEmit` 또는 `npm run build` — 타입 체크
2. `npm run lint` — 린트 체크
3. `npm run test` — 테스트 실행 (있는 경우)
4. 실패 시 에러 메시지 분석 후 원인 요약

---

## 결과 보고 형식

```
## 빌드/테스트 결과
- 컴파일: 성공/실패
- 테스트: X건 성공, Y건 실패, Z건 스킵
- 실패 원인: (있는 경우)
- 권장 조치: (있는 경우)
```
