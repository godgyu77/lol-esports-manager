# 보안 코딩 가이드라인

> AI 코드 생성 시 보안 취약점 방지를 위한 통합 보안 규칙이다.
> 프론트엔드/백엔드/공통 보안 규칙을 한곳에서 관리한다.
> 출처: ai-supplementary-rules 보안 섹션 + language-java-springboot-rules 보안 섹션 + final-review 보안 체크 통합

---

## 1. 프론트엔드 보안

### XSS (Cross-Site Scripting) 방지

- 사용자 입력을 **HTML에 직접 삽입 금지**
- `dangerouslySetInnerHTML` 사용 시 반드시 **sanitize** 처리 (DOMPurify 등)
- URL 파라미터, 쿼리스트링의 사용자 입력은 반드시 **검증/인코딩**
- 사용자 입력이 포함된 동적 HTML 생성 시 이스케이프 처리 필수

### 민감 정보 보호

- API 키, 시크릿을 **클라이언트 코드에 노출 금지** (환경 변수 + 서버 프록시 사용)
- 민감 정보가 브라우저 콘솔, 로컬 스토리지, 네트워크 탭에 노출되지 않는지 확인
- 에러 메시지에 내부 구현 세부사항 노출 금지

---

## 2. 백엔드 보안

### SQL 인젝션 방지

- **파라미터 바인딩 필수**: 문자열 연결(concatenation)로 SQL 쿼리 구성 금지
- ORM/쿼리 빌더 사용 시에도 raw query에 사용자 입력 직접 삽입 금지
- 동적 테이블명/컬럼명이 필요한 경우 화이트리스트 검증

### 인증/인가

- Spring Security 기반 인증/인가 구현
- 비밀번호: **BCrypt** 인코딩 (평문 저장 금지)
- CORS 설정: 허용 origin을 명시적으로 지정 (`*` 금지)
- 세션/토큰 관리 시 적절한 만료 시간 설정

### 에러 응답 보안

- 에러 응답에 민감 정보(DB 쿼리, 스택 트레이스, 내부 경로) **노출 금지**
- 프로덕션 환경에서 상세 에러 메시지 비활성화
- 일관된 에러 형식 사용, 적절한 HTTP 상태 코드 반환

### 입력 검증

- Controller 레벨: 기본 검증 (null, blank, 형식) — `@Valid`, Bean Validation
- Service 레벨: 비즈니스 규칙 검증
- 파일 업로드 시 **확장자 + MIME 타입 + 크기** 검증

---

## 3. 공통 보안

### 시크릿 관리

- 민감 정보(비밀번호, 토큰, 키)는 **로그에 출력 금지**
- `.env`, `application-secret.yml` 등 시크릿 파일은 **`.gitignore`에 반드시 포함**
- 하드코딩된 시크릿 금지 — 환경 변수 또는 시크릿 매니저 사용
- 커밋 전 시크릿 포함 여부 확인

### 의존성 보안

- 의존성 패키지의 **알려진 취약점** 정기 점검
  - 프론트엔드: `npm audit`
  - 백엔드: OWASP Dependency-Check, `mvn dependency:analyze`
- 취약점이 발견된 패키지는 가능한 빠르게 업데이트

### 데이터 보호

- 개인정보, 금융정보 등 민감 데이터는 암호화하여 저장
- 로그에 민감 데이터 마스킹 처리 (예: 카드번호 `****-****-****-1234`)
- 불필요한 데이터 수집/저장 최소화

---

## 4. 보안 체크리스트

> 코드 리뷰 및 최종 리뷰 시 확인 항목

- [ ] 사용자 입력에 대한 검증/sanitize가 있는가
- [ ] 민감 정보가 클라이언트/로그에 노출되지 않는가
- [ ] SQL 파라미터 바인딩을 사용하는가
- [ ] 하드코딩된 시크릿이 없는가
- [ ] `.gitignore`에 시크릿 파일이 포함되어 있는가
- [ ] 에러 응답에 내부 구현 정보가 노출되지 않는가
- [ ] CORS 설정이 적절한가
- [ ] 파일 업로드 시 검증이 있는가
