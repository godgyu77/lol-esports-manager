# AI 보완 규칙

## 환각 방지

- 존재하지 않는 API/메서드 생성 금지 — 사용 전 코드베이스에서 확인
- `package.json`에 명시된 버전의 API만 사용
- 불확실하면 "확인이 필요합니다"라고 명시
- import 모듈, 메서드, 타입, 파일 경로 존재 여부 확인 후 사용

## 리팩토링 원칙

- 동작 변경 없이 구조만 개선 (기능 변경은 별도 작업)
- 사용되지 않는 코드라도 삭제 전 사용자 확인
- 재사용 2회 미만인 로직의 공통화 금지
- 리팩토링과 기능 추가를 같은 커밋에 혼합 금지

## 코드 생성 품질

- 데드 코드(미사용 변수/import/함수) 없을 것
- 하드코딩된 환경/설정 값 없을 것
- 실패 가능한 연산(네트워크, 파일 I/O)에 에러 처리 필수
- null, undefined, 빈 배열 등 경계값 처리 확인
- 이벤트 리스너, 타이머, 구독의 cleanup 확인

## 코드 리뷰 트리거

- 공통 컴포넌트/유틸 수정 시 → 전체 사용처 영향도 분석 필수
- API 인터페이스 변경 시 → 프론트/백 양측 호환성 확인
