# 코드 품질 4원칙

## 1) 가독성

- 동시에 실행되지 않는 분기는 컴포넌트/함수로 분리
- 구현 상세는 이름 있는 함수/훅으로 추상화
- 조건식과 매직 넘버에 이름 부여
- 위에서 아래로 읽히도록 배치

## 2) 예측 가능성

- 함수/컴포넌트 이름과 반환 형태를 일관되게 유지
- 같은 역할의 훅/유틸은 입력/출력 타입 규약 통일
- 숨은 부수효과 줄이고 명시적 흐름으로 드러냄

## 3) 응집도

- 함께 수정되는 코드는 같은 디렉터리/모듈 경계에 배치
- 공유해야 하는 규칙만 공통화, 불필요한 공통화 금지
- 폼/상태/검증 로직은 분리하되 함께 바뀌는 단위는 묶음

## 4) 결합도

- 책임 단일화로 수정 영향 범위 최소화
- 과도한 추상화보다 제한된 중복 허용
- Props Drilling 과도 시 컨텍스트/상태 경계 재설계

## 적용 우선순위

- 위험도 높은 영역 → 응집도 우선
- 위험도 낮은 영역 → 가독성 우선
- 최적화보다 이해 가능한 구조 먼저
