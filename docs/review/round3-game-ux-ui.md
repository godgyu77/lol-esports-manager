## 1. 게임 시스템 밸런스 (Game Designer)

### 최우선 수정 3가지

| # | 항목 | 설명 |
|---|------|------|
| 1 | 승률 스노우볼 과잉 | 단일 게임 내 승률 이동 총량 +0.9 가능 → 클램프(0.85)에 항상 도달. 보정값 50% 하향 필요 |
| 2 | 경제 시스템 고정 지출 부재 | 수입 >> 지출 → 순흑자 구조. 시설비/코칭비 등 고정 지출 추가 필요 |
| 3 | 난이도 상수 중복 | difficultyEngine vs systemPrompt 서로 다른 값. 통합 필요 |

### 기타 밸런스 이슈

| 시스템 | 이슈 | 제안 |
|--------|------|------|
| 선수 성장 | 성장기 최대 +5 OVR/시즌 → 유망주 육성 4시즌 필요 | 17~19세 신인 폭발 성장(x1.3) 추가 |
| 선수 성장 | 전성기 변동 -0.5~+1.0 → 거의 고정 | 범위 -1.5~+2.0으로 확대 |
| 만족도 | 서포트 KDA 불리 → 항상 낮은 만족도 | 포지션별 기대 KDA 기준선 차등 |
| 만족도 | 개별 요소 20 미만이어도 불만 미생성 | 개별 요소 임계값 추가 |
| 메타 | 전략 효율 보정 +-0.01~0.1 → 영향 미미 | +-0.05~0.15로 상향 |
| 메타 | 적응 기간 7일 → 너무 짧음 | 14~21일 + 코치 능력 차등 |
| 시리즈 | 모멘텀 최대 +-4% → 리버스 스윕 거의 불가 | +-8%로 확대 |

## 2. UX 사용자 경험 (UX Researcher)

### 즉시 개선 (높은 영향)

| # | 문제 | 제안 |
|---|------|------|
| 1 | 시즌 진행 진입점 이중화 (모달 vs DayView) → 이벤트 놓침 | 모달에서 핵심 이벤트 요약 표시 또는 DayView로 통일 |

### 단기 개선

| # | 문제 | 제안 |
|---|------|------|
| 4 | 비경기일 지루함 — 훈련/휴식 반복 | 스킵 시 요약 리포트, 미니 이벤트 빈도 증가 |
| 5 | 튜토리얼 텍스트만 — 인터랙티브 가이드 없음 | UI 하이라이트 오버레이 + 단계별 안내 |
| 6 | Info 그룹 8개 과부하 | "상대분석/패치메타" → Matches, "커리어/업적/팀히스토리" → History로 분리 |
| 7 | 스킵 진행률 표시 없음 | 프로그레스 바/처리 일수 카운터 |

### 감정 곡선 분석

```
흥미 ████████████████████░░░░░░████████████████░░░████████████████
     |-- 게임 시작 --|-- 반복 구간 --|-- 경기일 --|평일|-- 시즌 종료 --|
```
- 비경기일 구간의 지루함이 가장 큰 UX 위험
- 경기일 몰입감(중계/채팅/전술)은 강점
- 성취감 즉시성 부족 (업적 달성 연출 없음)

## 3. UI 비주얼 디자인 (UI Designer)
