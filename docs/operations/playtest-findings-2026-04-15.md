# Playtest Findings (2026-04-15)

## 목적
- 실제 플레이 테스트 중 발견된 문제를 작업 가능한 형태로 정리한다.
- FM 스타일 기준에서 어긋나는 UX, 텍스트, 진행 흐름, 시스템 버그를 우선순위별로 묶는다.
- 이후 구현 시 이 문서를 기준으로 개별 티켓이나 체크리스트로 쪼갠다.

## 핵심 방향
- Football Manager처럼 `뉴스/인박스 중심으로 지금 해야 할 일`이 보여야 한다.
- 사용자는 여러 화면을 돌아다니며 판단하면 안 되고, `다음 버튼 -> 필수 처리 -> 진행` 흐름으로 가야 한다.
- 중복 버튼, 도우미 텍스트, 과한 개발자성 설명은 줄이고 게임 정보만 보여야 한다.
- 한글 깨짐, 이름 로컬라이징 오류, 포지션 표기 오류는 최우선으로 정리한다.

## P0 - 즉시 수정 필요

- [ ] 글자 깨짐 전면 수정
  - 현상: 한글도 영어도 아닌 깨진 문자열이 여러 화면에 노출됨.
  - 영향: 기본 가독성 붕괴, 게임 완성도 신뢰 하락.
  - 확인 대상: 전 화면 공통, 특히 `StaffView`, `ScoutingView`, 각종 패널/버튼/알림.
  - 조치 방향: 인코딩 문제 제거, 깨진 문자열 전수 교체, UTF-8 기준 통일.

- [ ] 선수/스태프 이름 한글화 오류 수정
  - 현상: `Arcane -> 에이알씨에이엔`처럼 음절 단위로 이상하게 변환됨.
  - 요구: 선수와 스태프 이름은 영어 원문으로 복원. 기존 데이터 참고.
  - 영향: 몰입감 저하, 실제 플레이 체감 매우 나쁨.

- [ ] 뉴스가 사라진 문제 복구
  - 현상: FM처럼 봐야 할 뉴스가 제대로 보이지 않음.
  - 요구: 뉴스가 다시 핵심 허브 역할을 해야 함.
  - 영향: 시즌 진행, 지시, 리텐션 모두 약해짐.

- [ ] 시즌 진행 차단 규칙 재설계
  - 현상: 지금은 해야 할 일을 명확히 알려주지 않고 진행도 불편함.
  - 요구: FM처럼 뉴스/인박스에서 필수 처리 항목을 보여주고, 해당 건을 처리하지 않으면 다음 진행이 되지 않게 해야 함.
  - 영향: 현재 코어 루프의 핵심 문제.

- [ ] 경기 종료 후 드래프트 진행 예외 처리 오류 수정
  - 현상: 경기 끝난 뒤 `드래프트가 먼저 필요하다`는 문구 때문에 진행이 막힘.
  - 기대: 피드백 화면 이후 자연스럽게 드래프트로 넘어가야 함.
  - 판단: 기존 예외처리가 잘못 들어간 것으로 보임.

## P1 - 플레이 흐름/UX 핵심 문제

- [ ] 화면 배치 불균형 수정
  - 현상: UI가 한쪽에 몰리거나 보기 불편한 배치가 있음.
  - 요구: FM처럼 한눈에 보기 쉬운 레이아웃.

- [ ] 시즌 진행 UI를 FM형 구조로 재설계
  - 현상: 여기저기 이동하면서 뭘 해야 할지 모름.
  - 요구: 뉴스/인박스에서 오늘 해야 할 일 확인 -> 처리 -> 다음 진행 버튼 흐름으로 정리.

- [ ] 좌측 하단 버튼 중복 검토
  - 현상: `팀 케미스트리 점검`이 좌측 하단과 우측 상단 등 중복 노출.
  - 요구: 좌측 하단은 `저장/불러오기`, `메인 메뉴` 정도만 두는 방향 검토.

- [ ] 우측 상단 진행 버튼을 FM 스타일로 정리
  - 요구: 우측 상단은 핵심적으로 `다음` 버튼 중심이어야 함.
  - 세부: 필수 업무 처리 전에는 잠기고, 처리 후에만 진행 가능하도록 검토.

- [ ] 개발자식 설명 문구 숨김
  - 현상: `OO 루프에서 전술 조정을...` 같은 개발 로그/설명성 문구가 노출됨.
  - 요구: 이런 문장은 유저에게 보이지 않게 제거.
  - 대상: 도움말, 안내 패널, 임시 설명 문구 전반.

- [ ] 즉시판단류 도우미 툴팁 제거 검토
  - 현상: 몰입보다 UI/UX를 해치는 방향으로 느껴짐.
  - 요구: FM처럼 정보는 자연스럽게 보이되, 과한 도우미 레이어는 축소.

- [ ] 전술 메뉴 중복 검토
  - 현상: 좌측 사이드 메뉴에 전술이 두 개로 보임.
  - 요구: 메뉴 구조 중복 여부 검토 및 단일화.

- [ ] 날짜 변경 로딩 연출 추가
  - 요구: 다음 진행 시 로딩창에서 날짜가 넘어가는 것이 보이면 좋음.
  - 목적: 시간 흐름 체감 강화.

## P1 - 게임 시스템/반응성 문제

- [ ] 스폰서 선택 후 제안 초기화 로직 수정
  - 현상: 스폰서 선택 후 다른 제안들도 남아 있거나, 리로드 후 처리 흐름이 불명확함.
  - 요구: 하나를 선택하면 기존 제안은 초기화되고, 다음 제안은 다시 받아야 함.

- [ ] 스폰서 선택 즉시 반응성 개선
  - 현상: 선택 후 금액 등 반영이 즉각적이지 않음.
  - 요구: 클릭 즉시 UI와 데이터에 반영되어야 함.
  - 비고: 웹 구조든 다른 언어든 성능/반응성 확보가 목표.

- [ ] 밴/픽 추천 품질 검토
  - 현상: 추천이 너무 이상하게 느껴짐.
  - 요구: 추천 근거, 챔피언 우선순위, 상대 조합 대응 로직 점검 필요.

- [ ] 상단 양팀 경기 정보 불일치 검토
  - 현상: 상단에 보여주는 양측 경기 정보가 실제와 다름.
  - 요구: HUD/상단 스코어보드 정보 정합성 점검.

## P1 - 라이브 매치 연출

- [ ] 해설 채팅 방향 수정
  - 현상: 해설진 채팅이 아래에서 나오고 흐름이 어색함.
  - 요구: 위에서 최신 대화가 쌓이는 방식으로 정리.

- [ ] 해설 텍스트 흐름 부드럽게 개선
  - 현상: 채팅이 부드럽게 올라오지 않음.
  - 요구: 자연스러운 갱신 템포와 누적 방식 필요.

- [ ] 해설진 말투 개성 강화
  - 현상: 해설진 특유의 말투가 잘 살아나지 않음.
  - 요구: 캐스터/해설 스타일 차이, 실제 중계 같은 어투 반영 검토.

## P2 - 로컬라이징/표기 개선

- [ ] 포지션 한글화
  - 요구: `top / jungle / mid / adc / support`를 `탑 / 정글 / 미드 / 원딜 / 서폿`으로 통일.

- [ ] 영어 유지가 더 나은 항목 재검토
  - 대상: 선수명, 스태프명 등 고유명사.
  - 방향: 이상한 음차 한글화보다 원문 영어 우선.

## 기술 검토 요청

- [ ] 현재 언어/웹 구조로 반응성 구현이 가능한지 검토
  - 이슈 제기: 웹이라서 즉각 반응이 어려운지 의문 제기됨.
  - 요구: 현재 구조로 해결 가능한지 먼저 판단하고, 구조 변경이 필요하면 그때 검토.
  - 비고: 바로 언어 전환을 결정하기보다 병목 지점 측정이 먼저 필요.

- [ ] 필요시 런타임/언어 재검토
  - 요청사항: 웹 구조로 반응이 안 되면 Python/C++ 등 다른 언어도 검토 가능하다는 의견.
  - 권장 접근: 전면 전환 전, 현재 병목이 렌더링/상태관리/DB/UI 구조 중 어디인지 진단 필요.

## FM 스타일 기준 요구사항 요약

- [ ] 뉴스/인박스가 메인 허브여야 함.
- [ ] 진행은 `다음 버튼` 중심으로 되어야 함.
- [ ] 해야 할 일을 처리하지 않으면 다음 진행이 막혀야 함.
- [ ] 여러 곳에 흩어진 CTA보다 한 군데서 명확히 지시해야 함.
- [ ] 개발자 설명이 아니라 게임 내 정보만 보여야 함.
- [ ] UI 구조는 FM 클론에 가깝게, 게임 내용만 다르게 느껴져야 함.

## 추천 작업 묶음

### 1차 묶음: 텍스트/표기 복구
- 글자 깨짐 수정
- 선수/스태프 영어 이름 복구
- 포지션 한글화
- 개발자식 설명 문구 제거

### 2차 묶음: 코어 진행 FM화
- 뉴스/인박스 허브 복구
- 필수 처리 전 진행 차단
- 우측 상단 `다음` 버튼 중심 구조
- 좌측 하단 버튼 정리

### 3차 묶음: 경기/라이브 품질
- 밴픽 추천 로직 점검
- 해설 채팅 방향/속도/말투 개선
- 상단 경기 정보 정합성 수정
- 경기 종료 후 드래프트 예외 처리 수정

### 4차 묶음: 반응성/시스템
- 스폰서 선택 즉시 반영
- 스폰서 제안 초기화 규칙 수정
- 날짜 진행 로딩 연출 추가
- 웹 구조 성능 병목 검토

- 다음 단계에서는 이 문서를 기반으로 `P0 -> P1` 순서로 실제 구현 백로그 문서로 다시 쪼개는 것이 좋다.
